# CleanSplit

CleanSplit is my shared-expenses app for the Spreetail assignment. I built it for the
flatmate scenario in the brief: Aisha, Rohan, Priya, Meera, Dev, and Sam need a cleaner
way to import a messy spreadsheet, review the problems, and understand who owes whom.

The app uses FastAPI, SQLite, SQLAlchemy, React, TypeScript, and Vite. I used AI as a
pair-programming assistant, mainly for scaffolding and first drafts, but I reviewed and
changed the important data-import and balance logic myself.

**AI used:** ChatGPT/Codex and Claude-style prompting for code generation, debugging, and
documentation drafts. The details, prompts, and mistakes are in `AI_USAGE.md`.

## What The App Does

- Login and register users.
- Create groups and add members with join dates and optional leave dates.
- Add expenses manually with equal, unequal, percentage, and share splits.
- Track USD expenses by storing the original currency and the INR value used for balances.
- Record settlements/payments between members.
- Import `expenses_export.csv` without editing it by hand.
- Stage every imported CSV row, show anomalies, and block commit until review rows are decided.
- Generate an import report from the same importer code used by the app.
- Show group balances plus per-member breakdowns so a user can trace how a number was built.

## Local Development

### Backend

On macOS/Linux:

```bash
cd backend
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt
export SECRET_KEY="local-dev-secret"
export CORS_ORIGINS="http://localhost:5173,http://127.0.0.1:5173"
uvicorn main:app --reload --port 8000
```

On Windows PowerShell:

```powershell
cd backend
python -m venv venv
.\venv\Scripts\Activate.ps1
pip install -r requirements.txt
$env:SECRET_KEY = "local-dev-secret"
$env:CORS_ORIGINS = "http://localhost:5173,http://127.0.0.1:5173"
uvicorn main:app --reload --port 8000
```

### Frontend

```bash
cd frontend
npm install
npm run dev
```

The frontend runs on http://localhost:5173 in development.

## Running The Assignment CSV

The provided CSV is committed at `data/expenses_export.csv`.

1. Register users for the flatmates.
2. Create a group, for example `Flat 4B`, with USD to INR rate `85`.
3. Add the group members with the correct membership dates:
   - Aisha, Rohan, Priya, Meera, Dev: `2026-02-01`
   - Sam: `2026-04-08`
   - Mark Meera left on `2026-03-31`
4. Open the group and choose Import CSV.
5. Upload `data/expenses_export.csv`.
6. Review the rows marked as needing review.
7. Approve or reject each review row, then commit the import.
8. Use the Download report button on the import review page to export the app-generated report.

To regenerate the checked-in baseline report from the command line:

```bash
cd backend
python generate_import_report.py ../data/expenses_export.csv --output ../IMPORT_REPORT.md
```

## Verification Commands I Used

Backend smoke test:

```bash
cd backend
python smoke_check.py
```

This test registers users, creates a group, uploads the unchanged CSV, checks expected
anomaly types, downloads the report endpoint, approves review rows, commits the import,
and verifies balances can be calculated.

Frontend production build:

```bash
cd frontend
npm run build
```

## Project Structure

```text
  models.py                  SQLAlchemy schema
  auth.py                    JWT auth helpers

  src/
    pages/

      ImportReview.tsx       row-by-row anomaly review and report download

```

## Notes And Limits

- SQLite is used because the assignment only requires a relational database and the
  dataset is small. The code uses `DATABASE_URL`, so moving to Postgres is possible.
- `IMPORT_REPORT.md` is a baseline report generated before reviewer decisions. The app
  can also download a report for a live import session after approvals/rejections.
- Deployment settings are included through Docker files, but the final deployed URL is
  separate from this repository setup.
